SpectreMind Project Handoff
Date: 2026-04-10

==================================================
1. What SpectreMind Is
==================================================

SpectreMind is a local-first, operator-controlled, session-based intelligence system for structured security work.

It is not meant to be a reckless autonomous hacking agent. It is meant to be a disciplined operator companion that helps with:
- interpreting requests
- planning actions
- enforcing scope and approval boundaries
- executing bounded local tools
- preserving evidence
- storing session memory
- surfacing useful follow-up suggestions
- generating clear reports

The design philosophy is simple:
SpectreMind should think with the operator, not instead of the operator.

Externally, the operator interacts with one system: SpectreMind.
Internally, SpectreMind uses specialized roles:
- AETHER = planner/orchestrator mind
- WATCHER = session intelligence and disciplined memory layer
- SCRIBE = reporting layer

Core doctrine that must remain intact:
- local-first
- explicit approval for execution
- single-target scope validation for operational actions
- evidence-driven outputs
- session-based workflows
- memory-aware but not noisy
- one unified external voice

==================================================
2. Current High-Level Architecture
==================================================

Main flow:
1. Operator sends a request
2. Intent parser handles hard-rule blocking first
3. Deterministic rule parsing catches explicit supported requests
4. Bounded LLM-assisted router handles ambiguous but allowed requests
5. SpectreMindCore normalizes and routes the request
6. Orchestrator builds and runs the operational path when applicable
7. Scope validation and approval gating occur before execution
8. Tool runs locally
9. Raw artifacts are saved
10. Parsed findings are stored
11. WATCHER processes structured results into session intelligence
12. SCRIBE generates report output
13. Chat messages and session state are persisted
14. SpectreMind returns one unified response to the operator

Key subsystems:
- app/core/spectremind_core.py = single external voice and main router
- app/core/orchestrator.py = operational execution spine
- app/core/intent_parser.py = rule-first parser plus bounded router fallback
- app/core/intent_router.py = LLM-assisted intent recovery for ambiguous but allowed phrasing
- app/core/memory_service.py = chat/session/long-term memory service
- app/core/scope.py = strict single-target validation
- app/agents/aether.py = planning and action selection
- app/agents/watcher.py = deterministic session intelligence
- app/agents/scribe.py = reporting
- app/storage/db.py = SQLite persistence layer
- app/tools/registry.py = structured tool registry
- app/tools/tool_cards.py = tool metadata system

4. Internal Agent Roles
==================================================

AETHER
Purpose:
- classify tasks
- choose action type
- route execution path
- remain bounded and not become a chaotic suggestion engine

WATCHER
Purpose:
- observe structured results
- maintain concise session intelligence
- generate deterministic suggestions based on real triggers
- preserve unresolved items and useful observations

SCRIBE
Purpose:
- convert structured findings into readable operator-facing reports
- summarize without inventing facts
- keep parser output as source of truth

==================================================
5. Important Safety and Control Principles
==================================================

These must remain intact:
- explicit approval before execution
- strict scope validation for operational actions
- no blind automation
- no invented targets or sessions
- no uncontrolled multi-step execution loops
- memory should remain disciplined, not bloated
- routing should fail closed when uncertain
- SpectreMind should stay unified externally even if internally multi-role

In plain terms:
usefulness should grow, but control must remain stronger than convenience.

9. What Must Not Be Lost In Future Work
==================================================

Do not regress these properties:
- one external voice through SpectreMindCore
- explicit approval before execution
- strict single-target validation for bounded operations
- evidence preserved locally
- parser output remains the source of truth for findings
- WATCHER remains disciplined and deterministic
- routing fails closed when uncertain
- memory remains concise and intentional
- tools remain operator-bounded, not autonomous

If these are weakened, SpectreMind will drift away from its goal.

==================================================
10. One-Paragraph Project Identity
==================================================

SpectreMind is a disciplined local intelligence system for authorized security work. It is designed to help an operator plan, validate, execute, remember, and report inside a session-based workflow while preserving control, scope, and evidence. It should become more fluent, more useful, and more capable over time, but never at the cost of operator control, bounded execution, or architectural clarity.
